🚀 Set Up Live Demo OPTIONAL

Connect your Gmail to see OmniTriage process real emails in real-time. Follow these 3 steps — the setup section will disappear once connected.
1
Create a Gmail App Password
▼

OmniTriage needs a Gmail App Password (not your regular password) to read and reply to emails securely.

  1. Go to Google App Passwords
  2. Sign in with your Google account
  3. Under "App name", type OmniTriage
  4. Click Create
  5. Copy the 16-character password (e.g. abcd efgh ijkl mnop)
💡 Prerequisite: 2-Step Verification must be enabled on your Google account. Go to Security → 2-Step Verification to enable it first.
2
Clone & Run Locally with Your Gmail
▼

Clone the repo and start the server with your Gmail credentials:

# Clone the repository git clone https://github.com/MANu13151/OmniTriageEnv.git cd OmniTriageEnv # Install dependencies pip install -r requirements.txt # Start with your Gmail credentials GMAIL_ADDRESS="your.email@gmail.com" \ GMAIL_APP_PASSWORD="xxxx xxxx xxxx xxxx" \ python server.py
⚠️ Replace your.email@gmail.com with your actual Gmail and xxxx xxxx xxxx xxxx with the App Password from Step 1.

The server will start on http://localhost:7860 and begin watching your inbox.

3
Send a Test Email & Watch It Get Triaged
▼

Send an email to the Gmail account you connected. Try these test scenarios:

🏢 Billing
Subject: "I was double charged"
🚨 Human Review
Subject: "My attorney will be contacting you"
🛡️ Fraud Block
Subject: "You won $1M lottery!"
👤 Ambiguous → Human
Subject: "Not sure if I should refund or return"

Within 10 seconds, the email will appear in the Live Email Inbox below, showing the triage result, department routing, and auto-reply sent.

💡 The system creates two Gmail labels automatically: Fraud-Quarantine (blocked scam emails) and Human-Review (emails needing human attention).
